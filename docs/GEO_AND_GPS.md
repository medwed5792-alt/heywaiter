# Геолокация и незаметный запрос GPS

## Настройка заведения

В документе **venues** добавлено поле **geo**:

```ts
geo: {
  lat: number,
  lng: number,
  radius: number  // метры, по умолчанию 100
}
```

Без `geo` проверка геозоны не выполняется.

---

## Как запрашивать GPS у гостя незаметно

1. **Не спрашивать сразу при открытии**  
   Запрос разрешения в момент входа в Mini App выглядит навязчиво и часто приводит к отказу. Лучше не вызывать `getCurrentPosition`/`watchPosition` в `useEffect` при первом рендере без контекста.

2. **Привязка к действию**  
   Запрашивать геолокацию **после первого осмысленного действия** пользователя: например, после нажатия «Открыть меню» или «Вызвать официанта». В этот момент уже есть контекст («заведение хочет предложить сервис»), и запрос воспринимается естественнее.

3. **Один запрос — дальше фон**  
   Один раз запросить позицию (или начать `watchPosition`) после действия. Дальше использовать тот же поток для фоновой проверки (интервал 30 с), не показывая повторных диалогов: браузер кэширует разрешение.

4. **Формулировка в коде**  
   В Mini App можно обернуть вызов в хук, который:
   - не стартует геолокацию при монтировании;
   - принимает флаг `startAfterUserAction: true` и запускает проверку после первого клика по кнопке (например, «Вызвать официанта»).

   Пример: при первом клике на любую кнопку вызова вызвать `startGeoFencing()`, затем уже использовать `useGeoFencing` с переданными venueId/tableId/sessionId.

5. **Fallback**  
   Если пользователь отклонил доступ — не показывать ошибку, просто не выполнять Geo-Fencing. Функциональность вызова официанта и меню остаётся.

Итого: **незаметно** = не при загрузке, а после первого действия гостя, один раз запросить доступ и дальше использовать его в фоне без повторных запросов.
